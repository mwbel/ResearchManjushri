## Main Takeaways

- 摘要 Harness 已成为产业共识,前沿模型持续逼近静态智能天花板。在此条件下,Harness 的经济价值由「扩展能力上限」转向「保障执行下限」:同一任务上,其对模型成绩的边际贡献随模型变强而系统性收窄,常见任务由弱模型时代的约 30% 压缩至强模型时代的约 7% 更长、更动态、更安全的前沿任务与跨重复执行的一致性是 Harness 的长存价值。如 OpenClaw 类的通用、全面框架，在Claude Code的日频更新冲击下，不再是创企机会 与此并行， Harness 对模型的工程化补强，直接利好模型在重端侧(
- 企业自有数据中心)与轻端侧(汽车、PC、手机)的本地部署 ：经工程化手段部署于企业自有数据中心的千亿参数模型，有望达到当下万亿参数旗舰模型的智能水平。而 企业仅承担电费与资本开支、并消除数据外泄风险 。这将打开覆盖企业本地推理与终端设备 AI 的庞大新市场,直接利好端侧硬件与本地部署生态 第一章 · Harness 的定义与构成 Harness 是位于模型与真实系统之间的工程执行层:模型提供概率性智能,Harness 提供将智能转化为可交付结果所需的上下文、工具、权限、执行环境、验证与审计。此定义在 2026 年已
- 为行业共识 [16][17]。
- Harness 由以下功能模块构成: 上下文装配(context engineering) : 决定放什么进上下文——检索、裁剪、排序。例:Anthropic context editing。 记忆与状态(memory & state): 跨轮次、跨会话的持久化与可恢复任务对象。例:Anthropic memory tool。 协议层(protocol) : 工具与数据的注册、调用与结果回填规范;各家一致或类似,MCP 已成事实标准。例:Model Context Protocol(MCP)。
- 接入与权限(access & authorization) : 接入网关与委托授权。例:GCP Agent Gateway(委托授权至 Cloud IAM)。 执行沙箱(execution sandbox) : 文件、网络、进程隔离的可销毁执行环境。例:Anthropic Managed Agents 自托管沙箱。 安全(security): 护栏、prompt-injection 防护、agent 身份、运行时隔离。例:Lakera;Palo Alto(收 Portkey)。
