摘要
Harness 已成为产业共识,前沿模型持续逼近静态智能天花板。在此条件下,Harness 的经济价值由「扩展能力上限」转向「保障执行下限」:同一任务上,其对模型成绩的边际贡献随模型变强而系统性收窄,常见任务由弱模型时代的约 30% 压缩至强模型时代的约 7%
更长、更动态、更安全的前沿任务与跨重复执行的一致性是
Harness 的长存价值。如 OpenClaw 类的通用、全面框架，在Claude Code的日频更新冲击下，不再是创企机会
与此并行，
Harness 对模型的工程化补强，直接利好模型在重端侧(企业自有数据中心)与轻端侧(汽车、PC、手机)的本地部署
：经工程化手段部署于企业自有数据中心的千亿参数模型，有望达到当下万亿参数旗舰模型的智能水平。而
企业仅承担电费与资本开支、并消除数据外泄风险
。这将打开覆盖企业本地推理与终端设备 AI 的庞大新市场,直接利好端侧硬件与本地部署生态
第一章 · Harness 的定义与构成
Harness 是位于模型与真实系统之间的工程执行层:模型提供概率性智能,Harness 提供将智能转化为可交付结果所需的上下文、工具、权限、执行环境、验证与审计。此定义在 2026 年已为行业共识 [16][17]。
Harness 由以下功能模块构成:
上下文装配(context engineering)
: 决定放什么进上下文——检索、裁剪、排序。例:Anthropic context editing。
记忆与状态(memory & state):
跨轮次、跨会话的持久化与可恢复任务对象。例:Anthropic memory tool。
协议层(protocol)
: 工具与数据的注册、调用与结果回填规范;各家一致或类似,MCP 已成事实标准。例:Model Context Protocol(MCP)。
接入与权限(access & authorization)
: 接入网关与委托授权。例:GCP Agent Gateway(委托授权至 Cloud IAM)。
执行沙箱(execution sandbox)
: 文件、网络、进程隔离的可销毁执行环境。例:Anthropic Managed Agents 自托管沙箱。
安全(security):
护栏、prompt-injection 防护、agent 身份、运行时隔离。例:Lakera;Palo Alto(收 Portkey)。
编排(orchestration,含 agent loop 与多 agent 调度)
: 计划—选工具—执行—观察—校验的闭环。例:Anthropic Claude Agent SDK。
可观测与审计(observability & audit):
意图、轨迹、审批、失败与恢复的记录。例:Claude Code session transcripts。
企业治理(enterprise governance)
: 身份、RBAC、配额、合规与数据边界。例:GCP Gemini Enterprise + Cloud IAM。
第二章 · Harness 对模型的边际贡献及其随时代的变化
2.1 弱模型时代(2025 年及之前):边际贡献显著
在通用助理任务上,裸模型基本无法独立完成。GAIA(通用 AI 助理的多步工具任务基准)上,无工具的 GPT-4o 仅个位数,接入通用 agent 框架(HuggingFace Transformers Agents、AutoGen 等)后升至 30% 以上,其中 HuggingFace Code Agent 验证集达 44.2% [9]。
Manus 是这一时代以 Harness 取得显著质量提升的代表。其系统由 Claude 叠加自有 context engineering、云端工具虚拟机与并行子 agent 构成,2025 年 3 月发布时在 GAIA 上 Level 1/2/3 分别为 86.5%/70.1%/57.7%,高于 OpenAI Deep Research 的 74.3%/69.1%/47.6%(Level 1 领先 12.2、Level 3 领先 10.1 个百分点)[8]。
同一模型更换 Harness,在高难度任务上的成绩摆动可达 30 个百分点以上。公开评测中,Opus 4.5 由通用 CORE-Agent scaffold 切换至 Claude Code,得分由 42% 升至 78%(+36);Cursor 的研究显示同一模型在不同 Harness 上为 46% 与 80%;CORE-Bench 在同模型、同后端、仅更换 Harness 时,分数移动 23.8 个百分点 [10]。
2.2 强模型时代(2026 年至今):边际贡献收窄,以 Fable 5 为例
随模型能力增强,更换 Harness 的成绩摆动显著钝化——从弱模型时代的约 30 分降至强模型时代的约 7 分 [10]。以 Opus 4.6 系统卡为例:同一 GPT-5.2-Codex 在 Terminus-2 harness 上 57.5%、在 OpenAI 自家 Codex CLI harness 上 64.7%,差 7.2 分(Terminal-Bench 2.0,890 trials)[1]。
Fable 5(2026-06-09,Mythos 级)较 Opus 4.8 在 SWE-bench Pro 提升约 11 分(80.3% vs 69.2%)[14][22],提升集中在跨阶段规划、依赖自治的委派、自写测试、视觉检测与工具调用——完成同类任务回合更少、速度快 25%–30% [22]。
新一代模型把原本由 Harness 补偿的编排路径缩短并内化。
但模型再强,
长程运行仍依赖 Harness 基底:Anthropic 称「将 Fable 放入 agent harness 方可连续工作数天」
[22],运行时 Harness 的必要性不减。
总结:
认知脚手架——规划、自检、工具选择,以及短期/单会话记忆——基本被模型逐渐内化(更长原生上下文与原生 memory tool)，
agent as tools或仍有空间
；仍然有价值且不被内化的是控制面：
跨会话与受治理/可携带的长期记忆
(模型状态、长期记忆必须由外部承担)、
执行沙箱与运行时
、
外部验证(全新上下文的验证 subagent 仍优于自我批评
[22])、
安全(护栏、prompt-injection 防护、agent 身份)、可观测与治理
。这些与模型质量正交,模型代际提升不予填补。
Harness 的补强空间等于模型的低分场景。

第三章 · 机会分析
3.1 大厂(云厂商)
已全面布局云、握有算力底座的大厂(AWS、Microsoft/Azure、GCP),正以 agent 为抓手推动 Agentic Cloud Transformation——把 Harness 并入既有云栈:第一方运行时与治理(AWS Bedrock AgentCore 的 Firecracker microVM 会话隔离 [15]、GCP Vertex AI Agent Engine 与 Agent Gateway [17]、Microsoft Agent 365 [18]),并吃下长上下文的硬件需求(KV 缓存随上下文线性增长、1M token 约 15GB/用户、可超过权重成为瓶颈,利好 HBM、高带宽显存与内存分层存储,NVIDIA ICMSP 将 KV 缓存在 HBM→CPU→NVMe 分层 [25])。
但优势仅限商品化层,三条划界:co-training 不外溢到与模型质量正交的治理/数据层;中立(跨模型、跨云)与商业立场结构性冲突,谁有立场谁做不了中立;客户私有数据不在云厂商侧(Google 自身定调亦为「模型在商品化,胜出者是数据架构最佳者」[17])。尤其跨云中立做不了——三大云互不信任、Agent 365 跨云仅 preview [18],该位置留给第三方(母本为 Snowflake、Datadog 在 AWS 体系内长成独立巨头)。
3.2 模型厂商(OpenAI / Anthropic / Google)
以模型为核心,把贴近模型、可联合训练的层做成自营产品(Claude Agent SDK、memory tool、Managed Agents),并以收购补齐 Harness 短板、中和潜在竞争。OpenAI 过去一年的 Harness 类收购最具代表性:Astral(uv/Ruff,开发者工具底座,并入 Codex)、Promptfoo(评测与安全)、Context.ai(评测,acqui-hire)、OpenClaw(agent 框架,acqui-hire);Anthropic 则收 Stainless(SDK 生成)、Bun(运行时)。边界明确:运行时资本密集且与模型质量正交,Anthropic 主动将执行沙箱外推给 Cloudflare、Daytona、Modal、Vercel [16];中立、跨云、客户数据与长程治理非其所长。
3.3 软件公司(数据与基础设施)
留存层天然附着于「客户数据 + 运行时 + 企业分销」,正是在位者的存量资产;它们不被云厂商替代的原因如下:
Snowflake、Databricks(数据 + 治理)。
三点使其不被任一朵云吞并:多云中立(同时跑在 AWS/Azure/GCP,云厂商无法强制迁移)、客户数据与血缘归属在其平台(Cortex Agents 在自家治理边界内访问数据、不外移,Horizon 审计;Databricks 接 Unity Catalog 列级血缘 [26])、数据重力带来的高迁移成本。云厂商的第一方件没有客户已治理的数据,替代不了。
Cloudflare(中立运行时 + 边缘 + 网关)。
不被云吞并的原因:它是跨云、跨模型的中立运行时
,模型厂商主动把执行层外包给它(Anthropic Managed Agents
[16][26]);全球边缘网络与零信任隔离难以复制,且
与三大云核心业务互补而非正面竞争
。
安全厂(护栏与 agent 安全)。
模型侧安全(拒答、对齐)由 labs 内化;但运行时与 agent 安全——prompt-injection 防护、护栏、agent 身份、工具调用授权、运行时隔离——与模型质量正交,属留存层。
独立玩家如 Lakera、Protect AI、HiddenLayer
[27];在位者经收购入场(Palo Alto 收 Portkey [19]、Promptfoo→OpenAI),Prompt Security 已由 Sentinel 运营,显示该层在快速整合。
其余受益软件按缺口分布:长程与状态→持久化执行与记忆(Temporal、Modal、mem0、Letta、Zep);一致性与实验室—生产 37% 落差→可观测、评测、护栏与 AI 保证(Arize、Langfuse、AIUC 类 [24])。
3.4 硬件公司(端侧)
Harness 以「多跑几遍 + 校验 + 编排」补偿较小模型的能力短板,使够用的中小模型在特定任务上达到可用结果,agent 负载因此可从超大规模云下沉至本地与边缘。各模型量级对应的智能水平与部署足迹如下(权重按 FP8 ≈ 1 字节/参数估,另加 KV 缓存,长上下文约 15GB/用户 [25];智能水平为约略对标)

3.5 创业公司
像 MindOS、OpenClaw 这类通用框架型产品已很难再出现——通用层被模型逐代内化、被厂商自营件替代,如今的增量在细分。真正的护城河是专有数据。
垂直应用的判定:做成「专有数据 × 工作流 × 监管/责任」闭环者不被模型替代,因模型触及不到行业私有数据——Legora(法律,约 55.5 亿美元)、Nabla(医疗,约 53 亿美元)、Hippocratic 即此类 [14];其在工具与工作流上的深度集成本身亦构成壁垒。缺数据闭环的「通用套壳」垂直工具,则可由模型加通用 Harness 复制。
结论
Harness 已成共识、模型持续变强,其经济价值由「扩展上限」转为「保障下限」,红利持续向长程、动态、可靠性与数据×工具迁移。沿模型同路竞速者(通用框架、通用 agent)终被追平或收购;赢面在模型不会经过的层——拥有客户数据、运行时与分销的横向在位者(Snowflake、Databricks、Cloudflare),以及以行业专有数据闭环成局的垂直应用。对创业者,可独立的路径是垂直数据闭环,而非更好的通用 agent。
硬件层面,Harness 的独特利好在端侧。模型自我进化拉动的是集中式训练算力(超大规模数据中心);而 Harness 配合够用的中小模型(70B–600B 量化后单台 8 卡节点即可承载,无需超大规模集群),使 agent 负载得以脱离超大规模云、下沉至企业本地与主权环境。这一路线直接利好端侧与本地硬件提供商——戴尔、HPE、Supermicro 等服务器 OEM,以及边缘与推理芯片。
