关于图片知识和视觉识别，可以这样理解：

**1. 图片在 RAG 里不能只存路径**  
如果只保存 `![](xxx.png)`，系统只知道这里有一张图，但不知道图里讲了什么，因此很难在检索和问答中真正利用图片。

**2. 更有用的做法是把图片变成“图片知识”**  
每张图片最好同时保存：
- 图片路径
- 所在章节和页码
- 图注或标题
- 图片前后的正文
- 必要时补一句这张图讲什么的说明

这样图片也能像文字一样进入知识库，被后续检索和回答使用。

**3. 所谓“图片内容”，就是图片表达的信息**  
例如它是一张流程图、示意图、表格截图、扫描页，图里有哪些文字，在说明什么主题，和前后正文是什么关系。

**4. 图片内容确实可以进一步解析**  
常见方法包括：
- OCR 识别图片中的文字
- 用视觉模型理解图片含义
- 结合图片、图注和上下文一起生成说明

**5. 视觉识别不必每次都做，完全可以作为可选功能**  
这也是更合理的做法。默认流程只处理文字和图片路径，需要时再单独对图片做视觉识别。这样有几个明显好处：
- 速度更快
- 成本更低
- 整体流程更稳定
- 不会因为图片分析拖慢整个 PDF 转 Markdown

**6. 最适合的设计方式是分成两层**
基础模式：
- PDF 转 Markdown
- 保留图片位置和图片文件
- 保存图注、页码、上下文文字

增强模式：
- 在基础模式之上
- 只对部分图片做视觉识别
- 补充“这张图讲什么”的说明

这样如果后续发现某些文档特别依赖图片内容，再开启增强模式即可，不需要一开始就对所有图片做复杂分析。

**一句话总结**  
最推荐的做法是先把图片作为“可检索的附加知识”保存下来，而把视觉识别作为按需开启的可选增强功能。
